## 11.1 Redis 缓存

### 用户信息缓存

```java
// JwtInterceptor.java
String cacheKey = "user:id:" + userId;

// 1. 先查 Redis 缓存
User user = (User) redisTemplate.opsForValue().get(cacheKey);

if (user == null) {
    // 2. 缓存未命中，查数据库
    user = userMapper.selectById(userId);
    if (user != null) {
        // 3. 回写缓存，TTL 30 分钟
        redisTemplate.opsForValue().set(cacheKey, user, 30, TimeUnit.MINUTES);
    }
}
```

**缓存策略**：
- **Key 设计**：`user:id:{userId}`（冒号分隔，便于管理）
- **TTL**：30 分钟（用户信息变化不频繁）
- **更新策略**：用户修改信息时删除缓存，下次访问自动回填

### 接口限流

```java
// RateLimitInterceptor.java
String key = "rate_limit:" + ip + ":" + uri;

// Redis 原子递增
Long count = redisTemplate.opsForValue().increment(key);
if (count == 1) {
    // 首次请求，设置过期时间（滑动窗口）
    redisTemplate.expire(key, rateLimit.time(), TimeUnit.SECONDS);
}

// 超过限制
if (count > rateLimit.count()) {
    throw new BusinessException(ErrorCode.SYSTEM_ERROR, "请求过于频繁");
}
```

**限流原理**：
- 使用 Redis 的 `increment` 原子操作
- 首次请求时设置过期时间（滑动窗口）
- 超过阈值则拒绝请求

### Redis 配置

```java
@Configuration
public class RedisConfig {
    @Bean
    public RedisTemplate<String, Object> redisTemplate(RedisConnectionFactory factory) {
        RedisTemplate<String, Object> template = new RedisTemplate<>();
        template.setConnectionFactory(factory);

        // JSON 序列化
        Jackson2JsonRedisSerializer<Object> serializer =
            new Jackson2JsonRedisSerializer<>(Object.class);
        ObjectMapper mapper = new ObjectMapper();
        mapper.setVisibility(PropertyAccessor.ALL, JsonAutoDetect.Visibility.ANY);
        serializer.setObjectMapper(mapper);

        template.setKeySerializer(new StringRedisSerializer());
        template.setValueSerializer(serializer);
        template.setHashKeySerializer(new StringRedisSerializer());
        template.setHashValueSerializer(serializer);
        template.afterPropertiesSet();

        return template;
    }
}
```
