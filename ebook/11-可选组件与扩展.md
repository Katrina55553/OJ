## 11.1 Redis 缓存

### 用户信息缓存

```java
// JwtInterceptor.java
String cacheKey = "user:id:" + userId;

// 1. 先查 Redis 缓存
User user = (User) redisTemplate.opsForValue().get(cacheKey);

if (user == null) {
    // 2. 缓存未命中，查数据库
    user = userMapper.selectById(userId);
    if (user != null) {
        // 3. 回写缓存，TTL 30 分钟
        redisTemplate.opsForValue().set(cacheKey, user, 30, TimeUnit.MINUTES);
    }
}
```

**缓存策略**：
- **Key 设计**：`user:id:{userId}`（冒号分隔，便于管理）
- **TTL**：30 分钟（用户信息变化不频繁）
- **更新策略**：用户修改信息时删除缓存，下次访问自动回填

### 接口限流

```java
// RateLimitInterceptor.java
String key = "rate_limit:" + ip + ":" + uri;

// Redis 原子递增
Long count = redisTemplate.opsForValue().increment(key);
if (count == 1) {
    // 首次请求，设置过期时间（滑动窗口）
    redisTemplate.expire(key, rateLimit.time(), TimeUnit.SECONDS);
}

// 超过限制
if (count > rateLimit.count()) {
    throw new BusinessException(ErrorCode.SYSTEM_ERROR, "请求过于频繁");
}
```

**限流原理**：
- 使用 Redis 的 `increment` 原子操作
- 首次请求时设置过期时间（滑动窗口）
- 超过阈值则拒绝请求

### Redis 配置

```java
@Configuration
public class RedisConfig {
    @Bean
    public RedisTemplate<String, Object> redisTemplate(RedisConnectionFactory factory) {
        RedisTemplate<String, Object> template = new RedisTemplate<>();
        template.setConnectionFactory(factory);

        // JSON 序列化
        Jackson2JsonRedisSerializer<Object> serializer =
            new Jackson2JsonRedisSerializer<>(Object.class);
        ObjectMapper mapper = new ObjectMapper();
        mapper.setVisibility(PropertyAccessor.ALL, JsonAutoDetect.Visibility.ANY);
        serializer.setObjectMapper(mapper);

        template.setKeySerializer(new StringRedisSerializer());
        template.setValueSerializer(serializer);
        template.setHashKeySerializer(new StringRedisSerializer());
        template.setHashValueSerializer(serializer);
        template.afterPropertiesSet();

        return template;
    }
}
```

## 11.2 Elasticsearch 帖子搜索

### 为什么需要 ES

```sql
-- MySQL LIKE 查询的问题
SELECT * FROM post WHERE content LIKE '%算法%';
-- 1. 全表扫描，性能差
-- 2. 不支持分词（"算法竞赛" 匹配不到 "算法"）
-- 3. 不支持相关性排序

-- ES 全文搜索的优势
-- 1. 倒排索引，毫秒级响应
-- 2. 中文分词（"算法竞赛" → ["算法", "竞赛"]）
-- 3. TF-IDF 相关性排序
```

### ES 集成


## 11.3 腾讯云 COS 文件上传

### 功能概述

- **用途**：用户头像上传
- **限制**：文件类型 jpeg/jpg/svg/png/webp，大小 ≤ 1MB
- **存储**：腾讯云对象存储（COS）

### 实现

```java
// CosClientConfig.java
@Configuration
public class CosClientConfig {
    @Value("${cos.accessKey}")
    private String accessKey;
    @Value("${cos.secretKey}")
    private String secretKey;
    @Value("${cos.region}")
    private String region;
    @Value("${cos.bucket}")
    private String bucket;

    @Bean
    public COSClient cosClient() {
        COSCredentials credentials = new BasicCOSCredentials(accessKey, secretKey);
        ClientConfig config = new ClientConfig(new Region(region));
        return new COSClient(credentials, config);
    }
}

// CosManager.java
@Component
public class CosManager {
    @Autowired
    private COSClient cosClient;

    @Value("${cos.bucket}")
    private String bucket;

    public void uploadFile(String key, InputStream inputStream) {
        ObjectMetadata metadata = new ObjectMetadata();
        cosClient.putObject(bucket, key, inputStream, metadata);
    }

    public String getFileUrl(String key) {
        return String.format("https://%s.cos.%s.myqcloud.com/%s",
            bucket, cosClient.getClientConfig().getRegion().getRegionName(), key);
    }
}

@PostMapping("/upload")

    // 4. 上传到 COS

}
```

## 11.6 如何启用可选组件

| 组件 | 启用条件 | 配置步骤 |
|------|----------|----------|
| **Redis 缓存** | 默认启用 | 配置 `REDIS_HOST`、`REDIS_PASSWORD` |
| **ES 搜索** | 需要 ES 实例 | 配置 `ES_URL`，启动 ES 服务 |
| **COS 上传** | 需要腾讯云账号 | 配置 `COS_*` 环境变量 |
| **微信集成** | 需要公众号 | 配置 `WX_*` 环境变量 |
| **JDoodle** | 需要 API Key | 配置 `JDOODLE_*` 环境变量 |

**Docker Compose 中添加 ES**：

```yaml
# 在 docker-compose.yml 中添加
elasticsearch:
  image: elasticsearch:8.10.0
  container_name: oj-es
  environment:
    - discovery.type=single-node
    - xpack.security.enabled=false
    - "ES_JAVA_OPTS=-Xms512m -Xmx512m"
  ports:
    - "9200:9200"
  volumes:
    - es-data:/usr/share/elasticsearch/data
  deploy:
    resources:
      limits:
        cpus: "2.0"
        memory: 1G
  networks:
    - oj-network
```
