## 2.2 本地开发环境搭建

### 2.2.1 数据库准备

#### MySQL

```bash
# 启动 MySQL 服务
# Windows：通过服务管理器或 net start mysql

# 登录 MySQL
mysql -u root -p

# 创建数据库
CREATE DATABASE yuoj DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

# 退出
exit

# 初始化表结构
mysql -u root -p yuoj < oj-backend/sql/create_table.sql
```

#### Redis

```bash
# 启动 Redis
redis-server

# 验证（另开终端）
redis-cli ping
# 返回 PONG 表示正常
```

#### RabbitMQ

推荐用 Docker 启动 RabbitMQ：

```bash
docker run -d --name rabbitmq \
  -p 5672:5672 \
  -p 15672:15672 \
  rabbitmq:3-management-alpine
```

管理界面：http://localhost:15672（guest/guest）

### 2.2.2 后端启动

```bash
# 进入后端目录
cd oj-backend

# 修改配置（可选，使用默认值即可连接本地服务）
# 编辑 src/main/resources/application.yml
# 确认以下配置：
#   spring.datasource.url: jdbc:mysql://localhost:3306/yuoj
#   spring.datasource.username: root
#   spring.datasource.password: 123456
#   spring.redis.host: localhost
#   spring.rabbitmq.host: localhost

# 编译并启动
mvn spring-boot:run
```

启动成功标志：

```
Started MainApplication in X.XXX seconds
```

验证后端：访问 http://localhost:8101/api/doc.html（Knife4j API 文档）

### 2.2.3 前端启动

```bash
# 进入前端目录
cd oj-frontend

# 安装依赖
npm install

# 启动开发服务器
npm run serve
```

启动成功标志：

```
App running at:
  - Local:   http://localhost:8080/
```

验证前端：访问 http://localhost:8080

### 2.2.4 开发环境代理说明

前端开发服务器通过 `vue.config.js` 配置了 API 代理：

```javascript
// vue.config.js
devServer: {
  port: 8080,
  proxy: {
    "/api": {
      target: "http://localhost:8101",  // 后端地址
      changeOrigin: true,
    },
  },
}
```

这意味着：
- 前端请求 `http://localhost:8080/api/user/login`
- 实际被代理到 `http://localhost:8101/api/user/login`
- 前后端端口不同，避免 CORS 问题

## 2.3 Docker 一键部署

### 配置环境变量

```bash
# 复制环境变量模板
cp .env.example .env

# 编辑 .env 文件，修改敏感配置（可选，有默认值）
# 主要关注：
#   DB_PASSWORD=your_password_here
#   JWT_SECRET=your-jwt-secret-key-at-least-32-chars
```

### 启动全部服务

```bash
# 一键启动（首次需拉取镜像，约 3-5 分钟）
docker compose up -d

# 查看服务状态
docker compose ps
```

预期输出：

```
NAME            STATUS          PORTS
oj-mysql        Up (healthy)    0.0.0.0:3306->3306/tcp
oj-redis        Up (healthy)    0.0.0.0:6379->6379/tcp
oj-rabbitmq     Up (healthy)    0.0.0.0:5672->5672/tcp, 0.0.0.0:15672->15672/tcp
oj-backend      Up              0.0.0.0:8101->8101/tcp
oj-frontend     Up              0.0.0.0:3000->80/tcp
```

### 服务启动顺序

```
MySQL ──┐
Redis   ├── healthcheck 通过 ──→ Backend ──→ Frontend
RabbitMQ┘
```

Docker Compose 通过 `depends_on` + `condition: service_healthy` 确保后端在数据库、缓存、消息队列都就绪后才启动。

### 访问服务

| 服务 | 地址 |
|------|------|
| 前端 | http://localhost:3000 |
| 后端 API 文档 | http://localhost:8101/api/doc.html |
| RabbitMQ 管理 | http://localhost:15672 |

## 2.4 环境变量详解

### 核心变量

| 变量 | 说明 | 默认值 | 必填 |
|------|------|--------|------|
| `DB_URL` | MySQL 连接地址 | `jdbc:mysql://localhost:3306/yuoj` | 否 |
| `DB_USERNAME` | MySQL 用户名 | `root` | 否 |
| `DB_PASSWORD` | MySQL 密码 | `123456` | **生产环境必改** |
| `REDIS_HOST` | Redis 地址 | `localhost` | 否 |
| `REDIS_PASSWORD` | Redis 密码 | 空 | 否 |
| `RABBITMQ_HOST` | RabbitMQ 地址 | `localhost` | 否 |
| `RABBITMQ_USERNAME` | RabbitMQ 用户名 | `guest` | 否 |
| `RABBITMQ_PASSWORD` | RabbitMQ 密码 | `guest` | 否 |
| `JWT_SECRET` | JWT 签名密钥 | 内置默认值 | **生产环境必改** |

### .env 文件格式

```bash
# 注释行以 # 开头
DB_PASSWORD=your_password_here    # 行内注释不支持
JWT_SECRET=your-secret-key-at-least-32-chars

# 值中包含空格需要用引号
SOME_VALUE="value with spaces"

# 空值表示不设置
REDIS_PASSWORD=
```

### 敏感信息处理建议

1. **开发环境**：使用默认值即可
2. **生产环境**：必须修改 `DB_PASSWORD` 和 `JWT_SECRET`
3. **Git 安全**：`.env` 已在 `.gitignore` 中，不会被提交
4. **团队协作**：共享 `.env.example`，各自创建 `.env`

## 2.5 常见问题排查

### 端口冲突

```
Error: Port 8080 is already in use
```

**解决**：找到占用端口的进程并关闭，或修改 `vue.config.js` 中的端口号。

```bash
# Windows 查找端口占用
netstat -ano | findstr :8080
taskkill /PID <PID> /F
```

### MySQL 连接失败

```
Communications link failure
```

**排查**：
1. MySQL 服务是否启动：`net start mysql`
2. 端口是否正确：默认 3306
3. 用户名密码是否正确
4. 数据库 `yuoj` 是否已创建

### RabbitMQ 连接超时

```
Connection refused: localhost:5672
```

**解决**：
```bash
# 用 Docker 启动 RabbitMQ
docker run -d --name rabbitmq -p 5672:5672 -p 15672:15672 rabbitmq:3-management-alpine

# 等待 30 秒后重试（RabbitMQ 启动较慢）
```

### Docker 权限问题

```
permission denied while trying to connect to the Docker daemon socket
```

**解决**（Linux）：
```bash
sudo usermod -aG docker $USER
# 重新登录
```

**解决**（Windows）：确保 Docker Desktop 已启动，且当前用户有权限。

### 后端启动报数据库错误

```
Access denied for user 'root'@'localhost'
```

**解决**：检查 `application.yml` 中的数据库密码是否与 MySQL 实际密码一致。默认是 `123456`。

### 前端 npm install 失败

```
npm ERR! code ERESOLVE
```

**解决**：
```bash
# 清除缓存重试
npm cache clean --force
npm install

# 或使用 --legacy-peer-deps
npm install --legacy-peer-deps
```
