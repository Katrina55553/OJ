## 3.3 权限控制

### 权限枚举

```typescript
// src/access/ACCESS_ENUM.ts
const ACCESS_ENUM = {
  NOT_LOGIN: "notLogin",  // 未登录（公开访问）
  USER: "user",           // 普通用户
  ADMIN: "admin",         // 管理员
};
export default ACCESS_ENUM;
```

三级权限层级：`NOT_LOGIN` < `USER` < `ADMIN`

### 权限校验函数

```typescript
// src/access/checkAccess.ts
import ACCESS_ENUM from "./ACCESS_ENUM";

const checkAccess = (loginUser: any, needAccess: string): boolean => {
  // 获取用户权限，默认为 NOT_LOGIN
  const loginUserAccess = loginUser?.userRole ?? ACCESS_ENUM.NOT_LOGIN;

  // 如果路由不需要登录，直接放行
  if (needAccess === ACCESS_ENUM.NOT_LOGIN) {
    return true;
  }

  // 如果路由需要用户权限，任何已登录用户都可以
  if (needAccess === ACCESS_ENUM.USER) {
    return loginUserAccess !== ACCESS_ENUM.NOT_LOGIN;
  }

  // 如果路由需要管理员权限，必须是管理员
  if (needAccess === ACCESS_ENUM.ADMIN) {
    return loginUserAccess === ACCESS_ENUM.ADMIN;
  }

  return true;
};

export default checkAccess;
```

### 全局路由守卫

```typescript
// src/access/index.ts
import router from "@/router";
import store from "@/store";
import ACCESS_ENUM from "./ACCESS_ENUM";
import checkAccess from "./checkAccess";

// 全局前置守卫
router.beforeEach(async (to, from, next) => {
  // 1. 获取当前用户信息
  //    首次访问时，会触发 API 调用获取用户信息
  let loginUser = store.state.user.loginUser;
  if (!loginUser || !loginUser.userRole) {
    // 尝试从后端获取用户信息
    await store.dispatch("user/getLoginUser");
    loginUser = store.state.user.loginUser;
  }

  // 2. 获取路由所需的权限级别
  const needAccess = (to.meta?.access as string) ?? ACCESS_ENUM.NOT_LOGIN;

  // 3. 如果需要登录但用户未登录
  if (needAccess !== ACCESS_ENUM.NOT_LOGIN && loginUser.userRole === ACCESS_ENUM.NOT_LOGIN) {
    // 重定向到登录页，并保存目标路径
    next(`/user/login?redirectUrl=${to.fullPath}`);
    return;
  }

  // 4. 权限校验
  if (!checkAccess(loginUser, needAccess)) {
    next("/noAuth");
    return;
  }

  // 5. 放行
  next();
});
```

### 权限校验流程图

```
用户访问 /admin
    │
    ▼
beforeEach 触发
    │
    ▼
获取 loginUser（可能触发 API 调用）
    │
    ▼
路由 meta.access = "admin"
    │
    ▼
用户未登录？ ──是──→ 重定向 /user/login?redirectUrl=/admin
    │
    否
    ▼
checkAccess(loginUser, "admin")
    │
    ▼
用户角色是 admin？ ──否──→ 重定向 /noAuth
    │
    是
    ▼
放行，渲染 /admin 页面
```

## 3.4 状态管理

本项目同时使用 **Vuex 4** 和 **Pinia**，这是一个常见的渐进式迁移策略：

| 状态管理 | 用途 | 原因 |
|----------|------|------|
| **Vuex** | 用户认证状态 | 项目初始化时使用，功能稳定 |
| **Pinia** | 主题切换状态 | 新功能使用，API 更简洁 |

### 3.4.1 Vuex 用户状态

```typescript
// src/store/index.ts
import { createStore } from "vuex";
import user from "./user";

export interface RootState {
  user: {
    loginUser: {
      userName: string;
      userRole: string;
      [key: string]: unknown;
    };
  };
}

export default createStore<RootState>({
  modules: { user },
});
```

```typescript
// src/store/user.ts
import { Module } from "vuex";
import { RootState } from "./index";
import { UserControllerService } from "../../generated";

const userModule: Module<any, RootState> = {
  namespaced: true,

  state: {
    loginUser: {
      userName: "未登录",
      userRole: "notLogin",
    },
  },

  actions: {
    async getLoginUser({ commit }) {
      try {
        const res = await UserControllerService.getLoginUserUsingGet();
        if (res.code === 0) {
          commit("updateUser", res.data);
        } else {
          // Token 过期或无效，清除本地 Token
          localStorage.removeItem("token");
          commit("updateUser", { userRole: "notLogin" });
        }
      } catch (error) {
        commit("updateUser", { userRole: "notLogin" });
      }
    },
  },

  mutations: {
    updateUser(state, payload) {
      state.loginUser = payload;
    },
  },
};

export default userModule;
```

### 3.4.2 Pinia 主题状态

```typescript
// src/store/theme.ts
import { defineStore } from "pinia";
import { ref, watch } from "vue";

export const useThemeStore = defineStore("theme", () => {
  // 初始化：优先读取 localStorage，其次检测系统偏好
  const isDark = ref<boolean>(
    localStorage.getItem("theme") === "dark" ||
      (!localStorage.getItem("theme") &&
        window.matchMedia("(prefers-color-scheme: dark)").matches)
  );

  // 切换主题
  const toggleDark = () => {
    isDark.value = !isDark.value;
    updateBodyClass();
    localStorage.setItem("theme", isDark.value ? "dark" : "light");
  };

  // 设置主题
  const setDark = (value: boolean) => {
    isDark.value = value;
    updateBodyClass();
    localStorage.setItem("theme", value ? "dark" : "light");
  };

  // 更新 body 的 arco-theme 属性
  const updateBodyClass = () => {
    if (isDark.value) {
      document.body.setAttribute("arco-theme", "dark");
    } else {
      document.body.removeAttribute("arco-theme");
    }
  };

  // 初始化时设置
  updateBodyClass();

  // 监听系统主题变化
  window.matchMedia("(prefers-color-scheme: dark)").addEventListener("change", (e) => {
    if (!localStorage.getItem("theme")) {
      isDark.value = e.matches;
      updateBodyClass();
    }
  });

  return { isDark, toggleDark, setDark };
});
```

### 双库共存的设计考量

```
为什么不用 Pinia 替换 Vuex？

1. 风险最小：Vuex 用户状态已稳定运行，替换需要全面测试
2. 渐进迁移：新功能用 Pinia，老功能保持 Vuex，逐步迁移
3. 学习成本：团队成员已熟悉 Vuex API，不需要一次性切换
4. 兼容性：Pinia 和 Vuex 可以完美共存，互不影响
```

## 3.5 API 层设计

### OpenAPI 自动生成

项目使用 **openapi-typescript-codegen** 工具，从后端的 Swagger/OpenAPI 规范自动生成 TypeScript API 客户端。

```bash
# 生成命令（需要后端服务运行）
openapi --input http://localhost:8101/api/v2/api-docs --output ./src/generated
```

生成的代码结构：

```
src/generated/
├── core/                    # 核心 HTTP 客户端
│   ├── ApiRequestOptions.ts
│   ├── ApiResult.ts
│   ├── AxiosHttpRequest.ts  # 基于 Axios 的请求封装
│   └── OpenAPI.ts           # 配置（baseURL、token）
├── services/                # 按 Controller 分组的服务类
│   ├── UserControllerService.ts
│   ├── QuestionControllerService.ts
│   └── ...
└── models/                  # TypeScript 类型定义
    ├── User.ts
    ├── Question.ts
    └── ...
```

### 服务类使用示例

```typescript
import { QuestionControllerService } from "@/generated";

// 获取题目详情
const res = await QuestionControllerService.getQuestionVoByIdUsingGet(id);
if (res.code === 0) {
```

### 后端响应约定

所有 API 响应遵循统一格式：

```typescript
interface BaseResponse<T> {
  code: number;    // 0 表示成功，非 0 表示错误
  data: T;         // 业务数据
  message: string; // 提示信息
}
```

### 直接 Axios 调用

对于非标准 REST 接口，直接使用 Axios：

```typescript

// 代码即时执行
```
